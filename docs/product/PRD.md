# PRD — tax-master × duty-save-frontend 병합 (절세 추천 단일 서비스)

> 상태: ready-for-agent (이슈 트래커 미설정 — 파일로 발행)
> 정답지: [TAX_SAVING.md](../domain/TAX_SAVING.md) v2 · 설계 근거: [DESIGN.md](../design/DESIGN.md)
> 도메인 용어: 그릇(Product) · 워터폴 · 트랜치 · 효율 · 확정/추진(Sourced) · 자격 게이트 · 긴급/전략 액션 · 점진공개 · 하이브리드 기본값

## Problem Statement

한국의 절세·자산형성 수단은 두 부류로 파편화돼 있다 — (1) **돈을 넣어 채우는 그릇**(연금저축·IRP·ISA·강화ISA·RIA·청년미래적금·해외주식 비과세 한도)과 (2) **상황에 따라 실행하는 절세 전략**(해외주식 손익통산·연도별 분산매도·가족 증여·고배당 한시 분리과세·금융소득 종합과세 관리). 입문자는 어느 그릇을 어떤 순서로 채울지도, 자기 상황에 어떤 전략이 유리한지도 한 번에 알기 어렵다.

현재 이 문제를 **두 개의 분리된 시제품**이 각각 절반씩만 풀고 있다:
- **tax-master** (Vite/React19): 월 적립을 그릇에 유리한 순서로 채우는 **워터폴**은 정확(트랜치 그리디, `Sourced` 확정/추진, 완전탐색 검증)하지만 상황별 전략이 없다.
- **duty-save-frontend** (Next14/React18): 손익통산·증여·고배당 같은 **전략 추천**(룰 기반 점수 top-5)과 풍부한 입력(가족·배당·보유현황)은 있으나, 그릇 적립 배분과 확정/추진 분리·검증된 계산이 없다.

사용자는 두 도구를 오가야 하고, 둘의 입력 폼은 길고 한 화면에 모든 걸 묻는다.

## Solution

두 시제품을 **하나의 서비스**로 합친다. 사용자는 **useFunnel 기반의 단계형 입력**으로 부담 없이 정보를 제공하고(점진공개), 결과로 **하나의 통합 추천**을 받는다:

- **📊 매달 적립 워터폴** — 그릇을 효율 순서로 한도까지 채우는 배분(tax-master 엔진).
- **⏰ 지금 당장 / 전략 액션** — 마감 임박 일회성(청년적금 D-day·RIA 절벽) + 상황별 전략(분산매도·가족 증여·고배당 분리과세·금소세 관리), 절세 효과 × 시급도로 정렬(duty 룰 패턴).

계산은 결정론적 순수 함수 하나(`recommend`)로 수렴하고, TAX_SAVING.md v2를 단일 정답지로 삼는다.

## User Stories

1. 재테크 입문자로서, 나이·연소득·월 투자가능액 3가지만 먼저 입력하고 즉시 추천 초안을 보고 싶다. 첫 화면에서 막히지 않기 위해.
2. 사용자로서, 입력을 한 화면에 몰아서 받지 않고 단계(퍼널)로 나눠 물어봐 주길 원한다. 긴 폼에 압도되지 않기 위해.
3. 사용자로서, 추가 정보를 넣을수록 추천이 더 정밀해지길 원한다(점진공개). 원하는 만큼만 깊이 들어가기 위해.
4. 직장인/프리랜서·사업/무소득을 토글로 고르고 그에 맞는 소득 기준이 적용되길 원한다. 자영업자도 정확한 판정을 받기 위해.
5. 사용자로서, 월 적립금이 어떤 그릇에 얼마씩·어떤 순서로 들어가는지 워터폴로 보고 싶다. "그래서 내 돈 얼마를 어디에"에 답을 얻기 위해.
6. 사용자로서, 각 그릇이 왜 그 순위인지 숫자(효율·첫 해 절세액)로 보고 싶다. 추천을 신뢰하기 위해.
7. 사용자로서, 각 그릇이 연 한도까지 얼마나 찼는지 시각적으로 보고 싶다. 남은 여력을 가늠하기 위해.
8. 청년(19~34세)으로서, 청년미래적금·청년형 강화ISA 자격과 등급(우대/일반/비과세만)을 안내받고 싶다. 정부지원을 놓치지 않기 위해.
9. 사용자로서, 청년미래적금 신청 마감(예: D-12)처럼 시한이 임박한 기회를 가장 먼저 보고 싶다. 기간을 놓치지 않기 위해.
10. 해외주식 보유자로서, RIA 감면율 절벽과 보유분 기준 예상 절감액을 안내받고 싶다. 지금 행동할지 판단하기 위해.
11. 해외주식 미실현 수익이 큰 사용자로서, 연도별 분산매도로 매년 250만 원 비과세를 활용하는 전략을 추천받고 싶다. 양도세를 0에 가깝게 줄이기 위해.
12. 같은 해에 수익·손실 종목을 가진 사용자로서, 손익통산 매도 전략을 안내받고 싶다. 과세표준을 낮추기 위해.
13. 미실현 수익이 큰 사용자로서, 가족 증여 후 매도 전략과 그 함정(증여 후 보유기간·부당행위계산 부인)을 경고와 함께 안내받고 싶다. 절세 효과가 무효화되지 않기 위해.
14. 고배당주 보유자로서, 2026~2028 한시 분리과세(일몰일 표시)를 추천받고 싶다. 종합과세 부담을 줄이기 위해.
15. 연 금융소득 2,000만 원에 근접한 사용자로서, 종합과세 회피 전략을 안내받고 싶다. 최고세율 과세를 피하기 위해.
16. 이미 금융소득종합과세 대상인 사용자로서, ISA·청년적금이 가입 불가임을 명확히 안내받고 연금/IRP 세액공제 위주 대안을 받고 싶다. 안 되는 걸 권하지 않게 하기 위해.
17. 본인의 금소세 대상 여부를 모르는 사용자로서, "비대상 가정"으로 일단 옵션을 보되 가정 배지로 표시받고 싶다. 옵션을 숨기지도 과대약속하지도 않기 위해.
18. 연금/IRP를 이미 일부 납입한 사용자로서, 보유·납입액을 입력해 잔여 한도 기준으로 추천받고 싶다. 중복 권유를 피하기 위해.
19. 단기에 쓸 돈(예: 3년)을 입력한 사용자로서, 55세까지 묶이는 연금/IRP가 워터폴에서 제외되고 그 이유를 안내받고 싶다. 자금이 묶이는 사고를 막기 위해.
20. 사용자로서, 입력 칸에서 기존 값 위에 바로 새 숫자를 입력하고(포커스 시 전체선택) 빈 칸도 허용되길 원한다. 0이 안 지워지는 불편을 겪지 않기 위해.
21. 고급 사용자로서, 국내/해외 기대수익률을 직접 조정해 운용절세 그릇의 순위 변화를 보고 싶다. 내 수익률 가정에 맞춘 결과를 얻기 위해.
22. 사용자로서, "이 시점에 시작한다면"으로 기준일을 바꿔 한시 상품·마감을 시뮬레이션하고 싶다. 시작 시점별 유불리를 비교하기 위해.
23. 사용자로서, 미확정(추진) 수치(강화ISA 한도/비과세 상향 등)는 계산에서 빠지고 정보로만 표시되길 원한다. 잘못된 숫자로 결정하지 않기 위해.
24. 사용자로서, 모든 추천 숫자에 법령 근거와 "자문 아님" 고지가 함께 보이길 원한다. 정보를 검증하고 한계를 이해하기 위해.
25. 사용자로서, 추천 결과를 별도 결과 화면(또는 라우트)에서 정리해 보고, 개별 항목을 더 깊이 볼 수 있길 원한다. 핵심을 먼저, 상세는 필요할 때 보기 위해.
26. 모바일 사용자로서, 퍼널·결과가 모바일에서 깔끔하게 동작하길 원한다. 휴대폰으로 끝까지 진행하기 위해.
27. 개인정보에 민감한 사용자로서, 자산·소득 정보가 서버로 전송되지 않고 브라우저에서 계산되길 원한다(현 단계). 프라이버시를 지키기 위해.
28. 개발자로서, 추천 로직 전체가 하나의 순수 함수 seam을 통과해 결정론적으로 테스트되길 원한다. 회귀 없이 세법 데이터를 갱신하기 위해.
29. 개발자로서, 향후 실제 백엔드로 전환할 때 비동기 어댑터 한 곳만 교체하면 되길 원한다. UI 변경 없이 백엔드를 붙이기 위해.
30. 운영자로서, 세법 개정 시 규칙 데이터(확정/추진·시행일·일몰일)만 교체하면 되길 원한다. 코드 수정 없이 매년 갱신하기 위해.

## Implementation Decisions

### A. 기술스택 = 현행 tax-master 유지 (Vite/React19/TS/Tailwind4)
**이 병합은 "기획의 통합"이지 기술 이전이 아니다.** 셸·빌드·디자인 시스템은 현행 tax-master를 그대로 쓴다. `duty-save-frontend`(Next14)는 **기능·전략·입력 기획의 출처(레퍼런스)** 로 취급하고, 그 안의 전략 룰·풍부한 입력·결과 구성을 tax-master 구조로 **재구현**한다. Next.js로 이전하지 않는다.
- duty에서 **가져오는 것(기획)**: 상황별 전략(손익통산·분산매도·가족증여·고배당·금소세 관리), 확장 입력 변수(보유현황·가족·배당·투자유형·미실현수익), 전략을 점수로 줄세우는 사고방식, 결과를 핵심→상세로 나눠 보는 구성.
- duty에서 **가져오지 않는 것(기술)**: Next App Router, 별도 라우팅 프레임워크, duty의 폼/디자인. (결과 상세 보기는 tax-master 내 뷰 전환/경량 라우팅으로 구현 — 구현 상세.)

### B. 단일 추천 엔진 = 워터폴(그릇) + 전략 룰(액션)의 합성
`recommend(profile, ruleSet, opts?) → Recommendation` 하나로 수렴한다.
- **그릇 배분**: tax-master의 결정론 트랜치 그리디 + `Sourced<T>` 확정/추진 게이트 + 공유풀(연금/IRP) + 락업 제외 + 자격 게이트/등급(variant) + 하이브리드 기본값. (TAX_SAVING §1–7)
- **전략·긴급 액션**: duty의 Rule 패턴(`isApplicable`/`evaluate` → 점수 = 절세효과 × 시급도)을 채택해 상황별 전략을 산출. 대상: 청년적금 D-day·RIA 절벽(기존 긴급 트랙 흡수) + 해외주식 손익통산·연도별 분산매도·가족 증여·고배당 한시 분리과세·금소세 관리·ISA→연금 전환. (TAX_SAVING §4-2~4-5, §8–10)
- 출력 `Recommendation` = `{ waterfall: Allocation[], actions: ActionCard[](긴급+전략, 점수순), assumptions: Badge[], disclaimers: string[], asOf }`.
- 점수 척도는 duty의 `computeScore(benefit, urgency)` / `URGENCY{IMMEDIATE,PARTIAL,STRUCTURAL,WARNING}` 패턴을 재사용.

### C. 통합 프로필 스키마 (zod, 정규화)
두 모델을 하나로 합치고 이름을 정규화한다. (수치 단위는 만원 입력 → 원 변환)
- **필수 (퍼널 1단계)**: `age`, `incomeType('earned'|'comprehensive'|'none')`, `income`, `monthlyInvestable`.
- **선택 (점진공개 후속 단계)**: `horizonYears`(기본 3, "언제 쓸 돈인지"=락업 하드 제약), 보유 계좌(`hasPension/Irp/Isa` + `pensionContribution/irpContribution`), `householdMedianPct`, `isFinanceTopTaxpayer`(또는 `financialIncome`로 도출), `investTypes[]`, `overseasUnrealizedProfit`(= 기존 `overseasHoldings`), `dividendIncome`, `holdsHighDividend`, 가족(`hasSpouse/hasChildren/hasMinorChildren`), `riskTolerance`(선택), `asOf`.
- 필드 충돌 정규화: `annual_salary`→`income`, `monthly_invest`→`monthlyInvestable`, `income_type` employee/freelancer/none ↔ earned/comprehensive/none, `foreign_stock_unrealized_profit`→`overseasUnrealizedProfit`.

### D. 입력 = useFunnel 다단계 퍼널 (DESIGN Q6 점진공개)
Vite/React 환경이므로 `@use-funnel/browser`(또는 react-router 어댑터)로 단계형 입력을 구성한다. 단계 구성(권장): ① 필수 3~4개 → (여기서 이미 결과 산출 가능) → ② 보유 계좌/잔여 한도 → ③ 투자 현황·해외주식 미실현 → ④ 소득·배당·가족. 각 단계는 최소 질문, 뒤로 가기/수정 가능. 퍼널 상태 → zod 파싱 → 정규 `UserProfile`. 미입력 항목은 하이브리드 기본값(게이트=포용, 금액등급=보수+업셀) + "가정했어요" 배지. (입력값 검증용으로 zod를 tax-master에 추가.)

useFunnel 퍼널 단계 형태(프로토타입에서 도출한 결정의 골격):
```
type Steps = {
  기본정보: { age?; incomeType?; income?; monthlyInvestable? }
  보유계좌: 기본정보 & { hasPension; pensionContribution?; hasIrp; irpContribution?; hasIsa }
  투자현황: 보유계좌 & { investTypes; overseasUnrealizedProfit?; horizonYears? }
  소득가족: 투자현황 & { financialIncome?; dividendIncome?; holdsHighDividend?; hasSpouse?; hasChildren?; hasMinorChildren? }
}
// 1단계 완료 후에도 결과 열람 가능: 이후 단계는 선택 심화.
```

### E. 규칙 데이터 = TAX_SAVING.md v2 단일 정답지
- 그릇 §1–7 → tax-master `Product`(Sourced 확정/추진, 시행일/한시 종료일). 강화ISA는 확정분(청년형 소득공제 200만)만 계산, 추진 수치는 `proposed` → 엔진 무시·정보 표시.
- 전략 §4-2~4-5, §8–10 → duty식 Rule(액션). 고배당 한시(§9)는 **일몰일 2028-12-31** 표기, 증여(§4-4)는 **보유기간 경고**(안전상 최장 10년 기준) 동반, 손익통산/분산매도는 결제일·연말(12/29) 주의 경고.
- 금소세(§10)는 자격 게이트 + 전략 양쪽에 반영.

### F. 엔진 seam + 비동기 어댑터
순수 함수 `recommend(profile): Recommendation`을 **유일한 핵심 seam**으로 둔다(결정론·테스트 대상). 그 위에 duty의 `fetchRecommendation(profile)` 비동기 어댑터를 유지해 데이터 경계를 한 곳에 모은다(향후 `fetch('/recommend')`로 교체). UI는 어댑터만 호출.

### G. 디자인 시스템 = 현행 tax-master "흐름" 유지
Pretendard + Space Grotesk, 세이지/골드/클레이 팔레트, 워터폴 시그니처(흐름 척추 + 한도까지 차오르는 골드 액체)를 그대로 사용. 새로 추가되는 전략 액션 카드는 같은 팔레트로 — 긴급=클레이, 구조적 전략=중립 — 시급도 색만 구분. Tailwind 4 `@theme` 그대로.

### H. 데이터/계산은 클라이언트 (현 단계)
개인정보 서버 미전송 유지(DESIGN Q7). 백엔드는 컴플라이언스 감사/프로필 저장 필요 시 도입(엔진 seam 그대로 이동 가능).

## Testing Decisions

좋은 테스트 = **외부 동작만** 검증(주어진 `profile` → 기대하는 워터폴/액션/배지), 내부 구현(트랜치 분해 방식, 점수 상수)에 묶이지 않는다.

- **핵심 seam `recommend()` (순수 함수)**: 단일 진입점에서 통합 결과를 검증. 기존 prior art = `tax-master/.../engine/__tests__/engine.test.ts` (세액공제 골든, 공유풀, 락업 제외, 긴급 D-day, RIA 절벽 추정, 등급/업셀, **그리디 최적성 완전탐색**, 고급모드 수익률). 이 스타일을 그대로 계승.
- **전략 룰**: 각 Rule의 `isApplicable`/`evaluate`를 골든 케이스로 — 예: 미실현 1,500만 → 분산매도 액션 등장 + 6년/0원, 금소세 대상 → ISA/청년적금 액션·그릇 비등장, 고배당 보유 → 한시 분리과세 액션(일몰 표시).
- **프로필 스키마**: zod 파싱 경계값(나이 19/80, 소득 0, 미입력→기본값) + 퍼널 상태→정규 프로필 매핑.
- **상호작용 회귀**: RIA 활성(보유 해외주식) → 신규 해외주식 직투 그릇 억제(기존 테스트 계승).
- 퍼널 UI 자체는 통합 동작(렌더·단계 전이)만 가볍게; 계산은 전부 `recommend()`에서 검증.

## Out of Scope

- 실제 백엔드/DB·로그인·프로필 영구 저장(어댑터 seam만 준비).
- 개별 종목·ETF 추천(그릇/전략 단위까지만, DESIGN Q17).
- 강화ISA **추진(미확정)** 수치 계산 반영(정보 표시만).
- ISA 농어민형, 근로청소년(15~19세) 자격.
- 기간말 다년 복리 최적화(현재 첫 해 절세액 기준; v1.1).
- 설명문구 LLM 윤색.
- 증여세·종합소득세 정밀 신고 시뮬레이션(전략은 안내·경고 수준).
