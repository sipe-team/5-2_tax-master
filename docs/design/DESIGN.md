# tax-master 설계 합의서 (v1)

> 본 문서는 그릴링 세션(18개 결정)으로 합의된 v1 설계 기준선이다.
> 규칙 수치의 정답지는 [`TAX_SAVING.md`](../domain/TAX_SAVING.md)이며, 본 문서는 *설계 결정*을 담는다.

## 한 줄 정의

한국 거주 개인의 **나이·연소득·월 투자가능액**을 받아, 7종 절세 그릇을
**"어떤 순서로 얼마씩 채워야 가장 유리한지"** + **"지금 당장 마감되는 기회"** 를
알려주는 *투명한 규칙 계산기* (자문 아님).

---

## 결과물 (Q1·Q9·Q17)

2단 출력:

- **⏰ 지금 당장** — 마감 임박 일회성 액션 (청년적금 D-day, RIA 감면 절벽). D-day·마감일 표시.
- **📊 매달 적립 워터폴** — 그릇 + 금액 + 순서 + *왜 이 순서인지 숫자로*.
  개별 종목은 다루지 않음 (자산군 카테고리 힌트만).

추천 단위 = **계좌·상품(그릇) + 금액 + 순서**. 개별 종목/ETF 추천은 범위 밖 (Q17).
청년적금·연금도 "투자 그릇"의 하나로 워터폴에 함께 포함.

---

## 계산 방식 (Q2·Q3·Q7·Q8)

- **결정론 TS 엔진, 프론트 단독** — 백엔드·LLM 없음, 개인정보 서버 미전송.
- **그리디**: "투입 1원당 세제혜택" 내림차순으로 한도까지 채움.
  상품 7종뿐이라 **완전탐색(brute force)으로 최적성 검증** (단위 테스트).
- **투트랙** (Q2):
  - v1 = **기본모드** (절세액 중심, 숨은 기본 수익률 **연 6%** 고정 — Q3).
  - 고급모드(해외/국내주식 수익률 직접 조정)는 v1.1.
- 설명가능성이 핵심 가치이므로 전역 최적화(블랙박스) 대신 그리디 채택.

## 상품 범위 (Q4·Q11·Q18)

| 상품 | MVP | 비고 |
|---|---|---|
| 연금저축펀드 | ✅ 전체 | IRP와 공유 풀 |
| IRP | ✅ 전체 | 연금과 합산 900만/1,800만 |
| ISA (현행) | ✅ 전체 | |
| 해외주식 직투 | ✅ 전체 | 연 250만 비과세 |
| RIA (한시) | ✅ 전체 | 감면율 날짜 절벽 → 긴급 트랙 |
| 청년미래적금 | ✅ 전체 | 신청 D-day → 긴급 트랙 |
| 강화ISA — 청년형 소득공제(연 200만, 확정) | ✅ | 적격 청년 워터폴 편입 |
| 강화ISA — 국민성장ISA 수치·국내투자형 등 | ❌ | `추진`(미확정) → MVP 제외 |

원칙: **"지금 확정된 것만 MVP"** (Q11).

## 규칙 데이터 (Q5·Q10·Q11)

- [`TAX_SAVING.md`](../domain/TAX_SAVING.md) = **source of truth**.
- 규칙은 JSON 데이터로 분리, 각 항목에 **법령 근거 · 시행일 · `확정/추진` 필드 태그**.
  엔진은 **`확정` 태그 필드로만 계산**, `추진`은 무시 (재사용 가능한 회색지대 처리 메커니즘).
- 상호작용 (Q10):
  - 연금 + IRP = **공유 풀** (연금 600 먼저 → IRP로 900까지).
  - RIA 추천 시 신규 해외주식 직투 **억제** (서로 깎아먹는 모순 추천 방지).
  - ISA → 연금전환 10%(최대 300만) 보너스는 **각주만** (미래·조건부, v1 계산 미반영).

---

## 입력 모델 (Q6·Q13·Q14·Q15·Q16)

첫 화면 **필수 3개**:
1. 나이
2. 연소득 — **직장인(총급여) / 사업·기타(종합소득) 토글**, 기본 직장인 (Q15)
3. 월 투자가능액

기본 가정: 기간 **3년**, 기준일 **오늘**.

**점진공개 (Q6)**: 나머지(종합소득·금소세 여부·가구중위소득·보유 해외주식)는 선택 입력.
안 넣으면 **하이브리드 기본값**으로 결과 산출 + "이렇게 가정했어요" 배지(탭하면 정정).

**하이브리드 기본값 (Q16)**:

| 미입력 항목 | 기본값 | 이유 |
|---|---|---|
| 금소세 대상 여부 (자격 게이트) | 비대상 가정 → 옵션 표시 (포용) | 대다수 비대상, 대상자는 본인이 앎 |
| 서민형/우대형 등 혜택 등급 (금액) | 낮은 등급(일반형) 가정 (보수) + 업셀 배지 | 과대약속 금지 |
| 가구중위소득(청년적금) | 일반형 tier 가정 + 단서 배지 | 개인소득 게이트가 주로 binding |

원칙: **자격 게이트는 포용적, 혜택 금액은 보수적+업셀.**

기타 입력 규칙:
- **자산(목돈) 입력 없음** (Q13) — 어떤 상품의 자격조건도 아님. 단 **보유 해외주식**은 RIA 조건부 입력으로만.
- **기간 = "언제 쓸 돈인지"(목표 시점)** (Q14) — 락업이 그걸 넘는 상품은 워터폴 제외 + 경고. 연금 락업 = (55 − 나이)년.
- 연봉과 **인과관계 없는 값(금소세·가구중위)은 가짜추정 안 함** (Q16) — 정직한 기본값+배지로 처리.

## 법적 성격 (Q12)

- "교육·정보 제공 도구"로 포지셔닝 ("맞춤 자문" 아님 — "투명한 규칙 계산기").
- 결과마다 **"정보 제공 목적, 투자·세무 자문 아님"** 상시 고지.
- 각 숫자에 **법령 근거 노출** (검증 가능성).
- 개인정보 서버 미전송(Q7)도 면책에 유리.

---

## MVP 경계 (Q18)

**v1 포함**
- 7종 전부 (파편화 해결 = 한눈에 비교)
- 기본모드 워터폴 (그리디 + 1원당 세제혜택)
- 2단 출력 + **긴급 트랙** ← 킬러 차별점
- 점진공개 입력(3개) + 하이브리드 기본값 + 가정 배지
- 자문 아님 면책 + 법령 근거 표시

**v1.1로 연기**
- 고급모드(수익률 직접 조정)
- 기준일 타임트래블 (v1은 오늘 고정)
- 자산군 카테고리 힌트 고도화
- 설명문구 LLM 윤색

## 열린 항목 (착수 전 확인)

1. **시의성 리스크** — 긴급 트랙이 의존하는 한시 창이 촉박: 청년미래적금 신청 **~7/3**, RIA 감면 절벽 임박. v1을 빨리 못 내면 후크의 시의성 약화 (= 지금 만들 이유이기도 함).
2. **컴플라이언스** — 토스 사내 프로젝트면 실제 리뷰 필요 여부 확인 → 면책 강도 조정.
3. **보수적 기본값의 정확한 값** — 원칙 합의됨. 실제 tier 값(가구중위 구간 등)은 `TAX_SAVING.md`에서 채움.

## 기술 스택

- React 19 + TypeScript + Vite (기존 스캐폴드)
- 백엔드 없음, 전부 브라우저 실행
- 규칙 = **타입 붙은 TS const 데이터**(`src/rules/`), 엔진 = 순수 함수, 검증 = 단위 테스트(완전탐색 포함)
  - JSON 대신 TS const 채택: 백엔드 없는 프론트 단독이라 컴파일 타임 타입검증이 가장 단순·안전(런타임 검증기 불필요). "규칙=데이터, 로직=엔진" 분리는 유지(데이터는 순수 const, import만).
  - 모든 법적 수치는 `Sourced<T>`로 감싸 `status(확정/추진)·근거·시행일` 부착 → 엔진은 `confirmed`만 계산.
